üìö Todo System API Documentation

üåê Servers

User Service: http://localhost:3001

Todo Service: http://localhost:3002

üîê Authentication

This API uses JWT Bearer Token authentication. After login, include the token in the Authorization header:

Authorization: Bearer <your-jwt-token>

üîë Authentication Endpoints

POST /api/users/register

Register a new user account

Request Body:

email
string (email format, required)
password
string (min 6 characters, required)

Responses:

201 - User registered successfully
400 - Invalid input data
409 - Email already registered

Example Request:

{
  "email": "user@example.com",
  "password": "password123"
}

POST /api/users/login

Login with existing credentials

Request Body:

email
string (email format, required)
password
string (required)

Responses:

200 - Login successful (returns JWT token)
401 - Invalid credentials

Example Response:

{
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "uuid": "123e4567-e89b-12d3-a456-426614174000",
    "email": "user@example.com"
  }
}

üìù Todo Endpoints

All todo endpoints require JWT authentication in the Authorization header.

GET /api/todos

Get all todos for the authenticated user

Responses:

200 - List of todos
401 - Unauthorized

Example Response:

{
  "todos": [
    {
      "id": 1,
      "user_uuid": "123e4567-e89b-12d3-a456-426614174000",
      "title": "Buy groceries",
      "description": "Milk, bread, eggs",
      "completed": false,
      "created_at": "2024-01-01T10:00:00Z",
      "updated_at": "2024-01-01T10:00:00Z"
    }
  ]
}

POST /api/todos

Create a new todo

Request Body:

title
string (max 255 chars, required)
description
string (max 1000 chars, optional)

Responses:

201 - Todo created successfully
400 - Invalid input data
401 - Unauthorized

Example Request:

{
  "title": "Buy groceries",
  "description": "Milk, bread, eggs"
}

PUT /api/todos/{id}

Update an existing todo

Path Parameters:

id
integer (todo ID, required)

Request Body:

title
string (max 255 chars, optional)
description
string (max 1000 chars, optional)
completed
boolean (optional)

Responses:

200 - Todo updated successfully
400 - Invalid input data
401 - Unauthorized
403 - Access denied
404 - Todo not found

Example Request:

{
  "title": "Buy groceries",
  "description": "Milk, bread, eggs, butter",
  "completed": true
}

DELETE /api/todos/{id}

Delete a todo

Path Parameters:

id
integer (todo ID, required)

Responses:

204 - Todo deleted successfully
401 - Unauthorized
403 - Access denied
404 - Todo not found

üìä Data Models

Todo Object

{
  "id": integer,
  "user_uuid": string,
  "title": string,
  "description": string,
  "completed": boolean,
  "created_at": string (ISO 8601),
  "updated_at": string (ISO 8601)
}

üîß Testing the API

You can test these endpoints using:

  • Postman: Import the api-collection.json file
  • cURL: Use the examples below
  • Frontend: Access the web interface at http://localhost:3000

Example cURL Commands:

# Register a user
curl -X POST http://localhost:3001/api/users/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'

# Login
curl -X POST http://localhost:3001/api/users/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password123"}'

# Create a todo (replace TOKEN with your JWT)
curl -X POST http://localhost:3002/api/todos \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{"title":"Test todo","description":"Test description"}'

# Get todos
curl -X GET http://localhost:3002/api/todos \
  -H "Authorization: Bearer TOKEN"
